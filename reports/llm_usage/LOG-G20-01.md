```text
# 4. Version Control Workflow

- **`main`** → Always deployable/stable. Protected branch. No direct commits.
- **`feature/*`** → All development (e.g., `feature/preprocessing`, `feature/qubo-solver`).
- **`bugfix/*`** → Hotfixes for evaluation bugs.

### Commit Conventions (Conventional Commits)
Adopt **`<type>(<scope>): <description>`** to auto‑generate changelogs if needed.
- `feat(preprocessing): add column dropping logic`
- `fix(model): handle missing target column`
- `test(qubo): add edge-case for allowance`
- `docs(readme): update installation steps`

### Pull Request Process
1. Push feature branch.
2. Open PR against `main`.
3. CI runs automatically (lint, typecheck, tests).
4. Merge only after CI passes and at least one teammate approves.

# 5. CI/CD Pipeline Outline (GitHub Actions)

Create `.github/workflows/ci.yml`:


name: CI Pipeline

on:
  push:
    branches: [ main, feature/*, bugfix/* ]
  pull_request:
    branches: [ main ]

jobs:
  lint-and-format:
    runs-on: ubuntu-latest
    steps:

      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install ruff

  type-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install mypy types-* # install relevant stubs
      - run: mypy src/ --ignore-missing-imports

  test:
    runs-on: ubuntu-latest
    needs: [lint-and-format, type-check]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'

# 7. Monitoring, Logging, and Error-Tracking

For a university project, we avoid heavy APM (e.g., Sentry/DataDog) to keep costs zero. Instead:

### Logging Strategy (Standard Library + Structured Format)
Create `src/qubo_project/utils/logger.py` (simplified):

import logging
import sys
from datetime import datetime

def setup_logger(name, log_file="logs/pipeline.log"):
    logger = logging.getLogger(name)
    logger.setLevel(logging.INFO)
    
    # Console handler (for CLI visibility)
    console = logging.StreamHandler(sys.stdout)
    console.setFormatter(logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s'))
    logger.addHandler(console)
    
    # File handler (for audit trail)
    file_handler = logging.FileHandler(log_file)
    file_handler.setFormatter(logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s'))
    logger.addHandler(file_handler)
    
    return logger

- **Log every mandatory function call** (start/end with duration).
- **Log α attempts** during QUBO search.
- **Log memory usage** (optional, via `memory_profiler` if needed).

### Error Tracking
- Use **Python's built-in `traceback`** to print full stack traces on exceptions.
- Wrap the CLI entrypoints in `try/except` that logs the error and exits with a non‑zero code (critical for automated evaluation).
- No external SaaS services.

**Justification**: Evaluators need to see why a pipeline failed. Structured logs to `/logs/` provide full traceability without over‑engineering.
```

```text
# 9. Justification Summary for Key Decisions
```
| Decision | Alternative | Why We Choose This |
| :--- | :--- | :--- |
| **`pandas` vs `polars`** | `polars` offers better memory usage. | `pandas` is the industry standard for academic projects; `pyarrow` backend closes the performance gap. The evaluation framework uses CSV, which `pandas` reads natively. |
| **`qubovert` vs `dimod`** | `dimod` is D‑Wave's standard. | `qubovert` is a pure‑Python, zero‑dependency package that handles dense matrices out‑of‑the‑box and is trivial to install. If it fails, we can implement our own Simulated Annealing in <100 lines. |
| **Streamlit vs Tkinter** | Tkinter is built‑in. | Streamlit requires zero HTML/CSS/JS, provides automatic state management, and can be run headlessly. The evaluator can easily launch it. A `requirements.txt` entry is simpler than fighting Tkinter platform issues. |
| **Docker vs virtualenv only** | Virtualenv is sufficient. | Docker guarantees that the Python version, system libraries (e.g., for `numpy`), and file permissions are identical on the evaluator's machine. The spec explicitly requires reproducibility (“riproducibilità”). |
| **GitHub Actions vs GitLab/others** | GitLab offers similar free tiers. | The repository is hosted on GitHub (mandated), so Actions are native, zero‑setup for tokens, and have 2000 free minutes/month—plenty for a university project. |
| **`ruff` vs `black` + `flake8`** | Black+Flake8 are the classic combo. | `ruff` is a single binary that is 10x faster and enforces both formatting and linting in one pass. This speeds up pre‑commit hooks and CI dramatically. |

```python
# src/qubo_project/utils.py
import logging
import sys
import json
from pathlib import Path
from datetime import datetime
from typing import Any, Dict

def setup_logger(name: str, log_file: str = "logs/pipeline.log") -> logging.Logger:
    """Configure a logger with both console and file handlers."""
    logger = logging.getLogger(name)
    if logger.handlers:  # avoid duplicate handlers if called multiple times
        return logger

    logger.setLevel(logging.INFO)

    # Ensure log directory exists
    Path(log_file).parent.mkdir(parents=True, exist_ok=True)

    # Console handler
    console = logging.StreamHandler(sys.stdout)
    console.setFormatter(logging.Formatter(
        '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
    ))
    logger.addHandler(console)

    # File handler
    file_handler = logging.FileHandler(log_file)
    file_handler.setFormatter(logging.Formatter(
        '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
    ))
    logger.addHandler(file_handler)

    return logger

def save_json(data: Dict[str, Any], filepath: str) -> None:
    """Safely write a dictionary to a JSON file."""
    Path(filepath).parent.mkdir(parents=True, exist_ok=True)
    with open(filepath, 'w') as f:
        json.dump(data, f, indent=2)

def load_json(filepath: str) -> Dict[str, Any]:
    """Load a JSON file into a dict."""
    with open(filepath, 'r') as f:
        return json.load(f)
```
